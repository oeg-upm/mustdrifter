    MuSTDrifter¶

    MuSTDrifter ¶

    Multi-Source Temporal Drifter for multidimensional discourse drift analysis.

    MuSTDrifter provides a unified interface to generate discourse representations, calculate drift between temporal periods, and produce reports and visualizations. It supports semantic, lexical, thematic, syntactic content, and syntactic style dimensions.

    Parameters:

    Name Type Description Default
    df DataFrame

    Input dataset containing the documents to analyze.

    		 required
    df_name str

    Dataset name used to define output paths and filenames.

    		 required
    results_path str

    Base directory where generated data, drift results, and reports are stored.

    		 required
    n_jobs int

    Number of parallel jobs used during computation. Defaults to 20.

    		 20
    K int

    Number of permutations or samples used in drift calculations. Defaults to 100.

    		 100
    device str

    Computational device used for processing, such as "cuda" or "cpu". Defaults to "cuda".

    		 'cuda'

    Base path for storing outputs.
    pos_annotations DataFrame

    POS annotations generated for the dataset.
    semantic_dimension DataFrame

    Semantic representation by period.
    lexical_dimension DataFrame

    Lexical distribution by period.
    thematic_dimension DataFrame

    Topic distribution by period.
    syntax_content_dimension DataFrame

    Syntactic content distribution by period.
    syntax_style_dimension DataFrame

    report_drift_tables dict

    Drift tables used for reporting.
    logger Logger

    The input dataframe is expected to contain at least a text column and a period identifier column. Generated dimensions and drift results are stored under the configured results_path.

    calculate_syntactic_style_drift(reference_period, test_period, metrics=['js_drift', 'kl_drift', 'log_drift'], rebase=None) ¶

    Calculate syntactic style drift between two periods.

    This method computes drift over syntactic style sub-distributions defined by UPOS contexts and aggregates the resulting drift scores across valid contexts.

    Parameters:

    Name Type Description Default
    reference_period str

    Reference period identifier.

    		 required
    test_period str

    Test period identifier.

    		 required
    metrics list

    Drift metrics used for syntactic style comparison. Defaults to ["js_drift", "kl_drift", "log_drift"].

    		 ['js_drift', 'kl_drift', 'log_drift']
    rebase bool

    Whether to recompute existing drift results. Defaults to None.

    		 None

    Returns:

    Type Description
    dict or None

    Drift results indexed by metric, or None if all requested results already exist and rebase is not True.
    Notes

    Syntactic style drift is computed independently for each valid context distribution and then aggregated using summary statistics such as mean, minimum, maximum, median, and standard deviation.

    calculate_drift(drift_dimensions=['semantic', 'syntactic_content', 'syntactic_style', 'lexical', 'thematic'], metrics=None, rebase=None) ¶

    Calculate drift for all ordered pairs of periods.

    This method iterates over all pairs of different periods in the dataset and computes drift for the selected dimensions.

    Parameters:

    Name Type Description Default
    drift_dimensions list

    Dimensions for which drift should be computed. Supported values are "semantic", "syntactic_content", "syntactic_style", "lexical", and "thematic". Defaults to all dimensions.

    		 ['semantic', 'syntactic_content', 'syntactic_style', 'lexical', 'thematic']
    metrics list

    Drift metrics to use. If None, default metrics are selected for each dimension. Defaults to None.

    		 None
    rebase bool

    Whether to recompute existing drift results. Defaults to None.

    		 None
    Notes

    Semantic drift uses ["cos_drift", "mmd_drift", "ks_drift"] by default. Lexical, thematic, syntactic content, and syntactic style drift use ["js_drift", "kl_drift", "log_drift"] by default.
